微信小程序

发表于 2018-06-13 |

什么是微信小程序、小程序的特点

使用场景：

  1. 业务逻辑简单
  2. 使用频率较低
  3. 性能要求低

小程序特点

  1. 不需要下载安装
  2. 用完即走，不需要安装过多应用
  3. 无处不在，随时可用

开发总体流程

注册小程序账号->配置小程序信息->开发->提交审核->发布

开发前期准备

  1. 注册小程序账号
    https://mp.weixin.qq.com/wxopen/waregister?action=step1
  2. 配置小程序信息
    https://mp.weixin.qq.com/wxopen/initprofile?action=index&lang=zh_CN&token=923936880
  3. 添加其他开发者
  4. 下载安装开发工具
    https://developers.weixin.qq.com/miniprogram/dev/devtools/devtools.html?t=2018413
  5. 新建模板项目

项目结构

wxml

wxml就是小程序页面的骨架，可以理解为web开发中的html，不同的是小程序里的标签对(组件)是自己跟html不完全一样。
常用的如下：

  1. view划分页面结构无实际意义，相当于div
  2. image 图片
  3. text 有它才能选择文字加样式
    模版语法

  4. 条件语句
    wx:if

  5. 循环
    wx:for

列表渲染 不写 item index 默认指代

  1. 数据绑定

wxss

  1. style
    可以直接在组件上写样式

  2. class
    在组件上引用wxss里定义的类样式

需要注意的是，小程序为我们预定义好了一个相对尺寸单位rpx，参考基准位750rpx，可以根据屏幕宽度进行自适应，我们只需要按照750px的设计稿将px按照1:1转换成rpx就可以了。

js

  1. 模块化 commonJs ES6 module
  2. 小程序运行时无Bom、Dom对象。
  3. 数据绑定 单向数据绑定 方法setData
  4. 事件支持
    https://developers.weixin.qq.com/miniprogram/dev/framework/view/wxml/event.html
    bindtap catchtap(阻止冒泡)

JSON 配置

app.json 是对当前小程序的全局配置，包括了小程序的所有页面路径、界面表现、网络超时时间、底部 tab 等.

API

微信相关原生能力APi挂在在wx对象上

  1. 网络请求
    wx.request(OBJECT)
  2. 缓存
    https://developers.weixin.qq.com/miniprogram/dev/api/data.html
  3. 页面跳转
    wx.navigateTo 父页面->子页面 有返回按钮 父页面只是被隐藏了 触发onUnload
    wx.redirectTo 跳向一个平行的页面 没有返回按钮 父页面被卸载了 触发onHide

小程序的组件化

  1. 利用预定义的组件
    https://developers.weixin.qq.com/miniprogram/dev/component/
  2. 自定义组件
    利用开发工具直接新建一个component
    需要使用自定义组件时需要在页面的配置文件申明

页面生命周期

  1. onLoad
  2. onShow
  3. onReady
  4. onHide
  5. onUnload

如何上线

开发完成后提交版本，经过腾讯审核后即可上线。
审核点注意点

  1. 取名
    首先，小程序的名称一定要与功能相关，不能模糊形容，不能以电话、邮件、日历等广义归纳类、普遍且不具有识别性词语来命名。
  2. 功能与类目要相符合
    小程序服务类目所对应的页面中的核心内容必须与该类目一致，例如，不能选择的类目是工具类，小程序里却搞起了电商。千万不能搞一些挂羊头卖狗肉的小程序，这是肯定要禁止的。
  3. 功能不能与微信官方的利息产生冲突
    微信官方有的功能，就不要碰了
  4. 不能含有诱导用户分享的内容
  5. 不得展示和推荐第三方小程序
  6. 必须登陆的小程序，必须提供测试账号
    该微信号将提供给微信审核人员审核微信小程序时登录使用，微信号需能够体验小程序的全部功能。
  7. 不能有bug，这是最基本的。

如何运维

微信公众平台提供有简单的客户端日志查询。

原生小程序缺点

  1. 不能直接使用npm的包，只能自己复制相关资源到小程序开发环境。
  2. 一套代码不能复用，由于小程序页面特有的语法，还需要为小程序单独开发，已有的vue项目可以快速迁移到小程序。
  3. 需要花更多的学习成本去学习小程序特有的组件等知识点。

mpvue

mpvue是什么？

mpvue 是一个使用基于Vue.js 开发小程序的前端框架。
http://mpvue.com/#_1

解决什么问题？

  1. 小程序引入了基于webpack的相关前端工程化工具链，带来了vue项目相同的开发体验。
  2. 已有的基于vue的项目更容易迁移到小程序，只需要改一些配置。

快速开始

用小程序开发工具打开项目即可

异步回调函数的缺点

  1. 书写顺序与代码执行顺序不一样，不利于维护与阅读
  2. 异步操作顺序发生变更的时候，需要不规模的代码重构
  3. 回调函数大多数都是匿名函数，导致bug不易追踪

Promise是什么

Promise是ES6中封装异步操作的规范，本身是一个构造函数。参数接收一个函数，该函数内部可以封装异步逻辑，通过改变promise的状态，来调用promise实例中的then或者catch方法。

面试问题总结

发表于 2018-04-22 |

一、http2.0

  1. 先说说http1.x的缺点：
    在http1.0的,再没开启connection：keep-alive的情况下。一次请求就会建立一个tcp连接，建立tcp连接是有延时的，因为需要经过三次握手。而浏览器对每个域名可创建的连接数是有限制的，因此资源的请求还需要排队（head of line blocking），同时，TCP连接会随着时间进行自我调谐，起初会限制连接的最大速度，如果数据传输成功，会随着时间的推移提高传输的速度，而http都是些短连接，tcp后续速度提升的特性不利用起来。这些归根结底就是没有复用连接，比较慢。
    http1.1默认开启connection：keep-alive，一定程度上复用了连接，但是短时间的复用，一般15s，同时一个连接里面也只允许一个连接和一个响应。

  2. http2.0
    http2就是针对http1.x的缺点而生的
    1）单一长连接、多路复用
    2）头部压缩、二进制格式
    3）服务端推送server push
